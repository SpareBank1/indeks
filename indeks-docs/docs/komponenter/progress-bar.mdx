# ProgressBar

ProgressBar viser fremdriften i en prosess — enten som en kontinuerlig progresjon (for eksempel opplasting) eller som status for fullføring (for eksempel steg ferdig/feilet). Komponenten gir brukeren tydelig feedback på hvor langt man har kommet, og om noe er fullført eller krever oppmerksomhet.

## Egnet til

- Brukeren må vente på at en prosess fullføres (lasting, opplasting, behandling)
- Det er viktig å vise fremdrift for å redusere usikkerhet
- En prosess har tydelig start og slutt
- Du vil bekrefte at et steg er fullført eller feilet
- Fremdrift er viktigere enn nøyaktig tidsestimat

**Eksempler:** opplasting av dokumenter, registrering/onboarding med bakgrunnsprosesser, systemprosesser som validering eller synkronisering.

## Uegnet til

- Prosessen er tilnærmet umiddelbar
- Du ikke kan vise reell eller meningsfull fremdrift
- Du kun vil indikere at noe pågår — bruk `Spinner` i stedet
- Du vil vise plassering i et flerstegsforløp — bruk Stepper
- Status handler om varsling eller tilbakemelding — bruk Message

## Retningslinjer

### Typer

ProgressBar finnes i to varianter:

- **Uten label** — brukes når kontekst er tydelig og ekstra forklaring ikke er nødvendig. Legg da til `aria-label` på elementet for tilgjengelighet.
- **Med label** — anbefalt når brukeren trenger forklaring på hva som skjer eller hva fremdriften gjelder.

### Prosent og verdi

Prosentvis verdi kan vises som del av label eller støttetekst dersom det gir reell verdi for brukeren. Unngå prosent hvis verdien er usikker eller beregnes kunstig. Ved ukjent varighet bør du vurdere en `Spinner` i stedet.

### Label og støttetekst

Label skal beskrive hva som skjer, ikke bare at noe pågår. «Laster opp dokumenter» er bedre enn «Prosessering». Unngå tekniske formuleringer.

Ved error skal label eller støttetekst forklare hva som gikk galt og hva brukeren kan gjøre videre. Ikke bruk støttetekst som bare gjentar labelen.

| Tilstand | God støttetekst | Dårlig støttetekst |
|----------|-----------------|--------------------|
| success | «Alle filer er lastet opp» | «Fullført» |
| error | «Opplastingen feilet. Sjekk filstørrelse og prøv igjen.» | «Noe gikk galt» |

### Én prosess om gangen

Ikke bruk flere ProgressBars for å vise deltilstander i samme prosess. Én sammenhengende prosess — én ProgressBar. Ikke bruk ProgressBar og Spinner for samme prosess.

### Bruk i flyt

Plasser ProgressBar nær innholdet eller handlingen den hører til. Unngå å bruke flere ProgressBars samtidig på samme side.

## Tilpasning med CSS

Trenger du `ix-progress-bar`-stylingen uten React-komponenten eller `<ix-progress-bar>` web component, kan du bruke klassene direkte. Merk at du da selv er ansvarlig for ARIA-attributter og label-kobling.

### Tilgjengelige klasser og custom properties

| Element / custom property | Beskrivelse |
|---------------------------|-------------|
| `.ix-progress-bar` | Rotelementet (grid-layout) |
| `.ix-progress-bar > label` | Labeltekst (kolonne 1) |
| `.ix-progress-bar__percent` | Prosentvisning øverst til høyre (injisert av web component, kun i aktiv tilstand) |
| `.ix-progress-bar > progress` | Nativt progress-element (full bredde) |
| `--ix-progress-bar-fill` | Farge på progress-verdien. Settes automatisk basert på `state`. |
| `[state="success\|error"]` | Settes på rotelementet. Styrer farge via `--ix-progress-bar-fill`. |

For å style progress-verdien direkte kan du bruke pseudo-elementene `::-webkit-progress-value` (Chrome/Safari) og `::-moz-progress-bar` (Firefox).

### Eksempel: ren HTML uten web component

```html
<div class="ix-progress-bar">
  <label for="pb-1">Laster opp dokumenter</label>
  <progress id="pb-1" value="65" max="100"></progress>
</div>
```

Uten `<ix-progress-bar>` web component injiseres ikke `__percent`-span eller statusikon automatisk. Nativt `<progress>` har `role="progressbar"` og `aria-value*` innebygd i moderne nettlesere.

## Universell utforming

### Hva du selv må sørge for

- **Gi komponenten et tilgjengelig navn** — bruk `<label>` som barn, eller legg til `aria-label` direkte på `<ix-progress-bar>` når synlig label mangler
- **Skriv meningsfull støttetekst ved error** — teksten annonseres til skjermlesere via `aria-live`. Forklar hva som gikk galt og hva brukeren kan gjøre videre
- **Ikke oppdater fremdrift for raskt** — hyppige endringer i `value` kan forstyrre skjermleserbrukere. Oppdater med intervaller som gir meningsfull informasjon

### Hva komponenten gjør automatisk

Når du bruker `<ix-progress-bar>` eller React-komponenten:

- Nativt `<progress>`-element gir `role="progressbar"`, `aria-valuemin="0"`, `aria-valuemax="100"` og `aria-valuenow` automatisk
- I success/error settes `aria-hidden="true"` på `<progress>` — statusen er ikke lenger en fremdriftslinje
- State-overganger annonseres via `aria-live="polite"` med «Fullført» eller «Feilet»
- Statusikon injiseres automatisk ved success og error med `aria-hidden="true"` — status formidles via tekst, ikke ikon alene
- `<label>` kobles til `<progress>` via `for`/`id` automatisk (genererer ID om nødvendig)
- `aria-label` på host videresendes til `<progress>` når synlig label mangler
