# DateField

DateField lar brukeren taste en dato i norsk format (`dd.mm.åååå`) eller åpne enhetens innebygde kalender via en knapp i feltet. Verdien som sendes videre er alltid ISO (`åååå-mm-dd`), uavhengig av hva som vises i feltet.

:::info Norsk format, ISO-verdi
Brukeren ser og taster `dd.mm.åååå`. Selve verdien — det som sendes ved innsending og gjennom `onChange` — er ISO (`åååå-mm-dd`). Det unngår den amerikanske `mm/dd/åååå`-tvetydigheten samtidig som datoen er maskinlesbar.
:::

## Egnet til

- Datoer brukeren kjenner og kan taste raskt (fødselsdato, startdato)
- Situasjoner der både tasting og en kalender bør være tilgjengelig
- Felt som trenger en nedre og/eller øvre grense for gyldige datoer

## Uegnet til

- Fritt datointervall der bruker velger «fra» og «til» (bruk to felter)
- Valg av måned eller år alene uten dag
- Sperring av enkeltdager eller helger — støttes ikke i denne fasen

## Kom i gang

I feltet taster brukeren datoen, som formateres fortløpende til `dd.mm.åååå`. Kalenderknappen til høyre åpner enhetens innebygde datovelger. På desktop uten `showPicker()`-støtte fanges klikket av en gjennomsiktig native date-input som ligger oppå knappen, så kalenderen åpnes robust helt ned til nettleser-baselinen vår.

<Tabs groupId="variant">
<TabItem value="html" label="HTML">

```jsx live
<ix-field>
  <label for="fodt-html">Fødselsdato</label>
  <span data-field="description">dd.mm.åååå</span>
  <ix-date-field class="ix-date-field" data-open-label="Åpne kalender" name="fodt">
    <div class="ix-text-field">
      <input id="fodt-html" inputmode="numeric" data-format="date" />
    </div>
  </ix-date-field>
  <span data-field="error" aria-live="polite"></span>
</ix-field>
```

Forfatteren skriver kun `.ix-text-field` med den synlige inputen (`data-format="date"`). Web-komponenten `<ix-date-field>` genererer selv kalenderknappen og den overlagte native date-inputen, flytter `name` til den native inputen (så innsending gir ISO), og synkroniserer de to veier.

</TabItem>

## Eksempler

### Forhåndsvalgt verdi

Startverdien settes i ISO (`åååå-mm-dd`) og vises som `dd.mm.åååå` i feltet. I React er `defaultValue` ukontrollert startverdi; `value` (sammen med `onChange`) gir et kontrollert felt.

<Tabs groupId="variant">
<TabItem value="html" label="HTML">

```jsx live
<ix-field>
  <label for="fodt-preselect">Fødselsdato</label>
  <span data-field="description">dd.mm.åååå</span>
  <ix-date-field class="ix-date-field" data-open-label="Åpne kalender" name="fodt" value="1990-05-17">
    <div class="ix-text-field">
      <input id="fodt-preselect" inputmode="numeric" data-format="date" />
    </div>
  </ix-date-field>
  <span data-field="error" aria-live="polite"></span>
</ix-field>
```

`value="1990-05-17"` på `<ix-date-field>` seeder både den synlige inputen (17.05.1990) og den native ISO-inputen.

</TabItem>

## Native skjema-innsending

DateField sperrer ikke for vanlige HTML-skjemaer. `name` når den genererte native date-inputen, som bærer ISO-verdien — så et server-side skjema med `method="post"` mottar `åååå-mm-dd`:

```html
<form method="post" action="/registrer">
  <ix-field>
    <label for="fodt-form">Fødselsdato</label>
    <span data-field="description">dd.mm.åååå</span>
    <ix-date-field class="ix-date-field" data-open-label="Åpne kalender" name="fodt">
      <div class="ix-text-field">
        <input id="fodt-form" inputmode="numeric" data-format="date" />
      </div>
    </ix-date-field>
  </ix-field>
  <button type="submit">Send</button>
</form>
```

## Retningslinjer

### Vis alltid formatet som tekst

Skriv `dd.mm.åååå` i `description`. Formateringen vises først når brukeren begynner å taste, så den er ikke et fullgodt format-hint alene. Ikke bruk placeholder til dette.

### Bruk min og maks fremfor etterkontroll der det går

Når gyldige datoer har en naturlig grense (f.eks. ikke frem i tid), sett `min`/`max` slik at OS-kalenderen sperrer dem. Regler som ikke kan uttrykkes med et intervall — enkeltdager, helger — valideres på blur/innsending med en `errorMessage`.

### Én dato per felt

Trenger du et intervall, bruk to separate datofelter med hver sin label («Fra» og «Til»).

## Kjent begrensning (fase 1)

Denne versjonen støtter `min`/`max`, men ikke sperring av enkeltdager eller helger, og bruker enhetens innebygde kalender. En egen kalendervelger for desktop med fler-års-navigasjon er planlagt i en senere fase. På eldre nettlesere uten `showPicker()` åpnes kalenderen fortsatt via den gjennomsiktige native date-inputen som ligger oppå knappen.

## Universell utforming

### Hva du selv må sørge for

- **Skriv en beskrivende label.** «Fødselsdato», ikke «Velg dato». Alle datofelter må ha en label, synlig eller via `ariaLabel`.
- **Oppgi formatet som synlig tekst** i `description` (`dd.mm.åååå`) — ikke som placeholder.
- **Send inn oversatt `openLabel`** (HTML: `data-open-label`). Det blir kalenderknappens tilgjengelige navn og har ingen innebygd fallback.
- **Skriv gode feilmeldinger.** Formateringen validerer ikke; valider på blur/innsending og forklar hva som er galt.

### Hva komponenten gjør automatisk

- Kobler label til den synlige inputen via `for`/`id`, og hjelpetekst/feilmelding via `aria-describedby` (ix-field).
- Genererer en `<button>` for kalenderen med tilgjengelig navn fra `openLabel`, og en `aria-hidden` native date-input som bærer ISO-verdien.
- Reserverer plass til kalenderikonet i feltet og speiler `min`/`max`/`disabled`/`readonly` til den native inputen.
- Synkroniserer den tastede `dd.mm.åååå`-verdien og den native ISO-verdien begge veier, og emitter én `change` per faktisk endring.
- På touch-enheter skjules kalenderknappen fra skjermleser slik at tap på den native inputen er primær interaksjon.

## Tilpasning med CSS

Trenger du `ix-date-field`-stylingen på HTML du setter sammen selv — uten React-komponenten — bruker du klassene direkte. Den synlige inputen gjenbruker `.ix-text-field`; kalenderknappen og den native inputen genereres av web-komponenten og trenger ikke skrives for hånd.

### Tilgjengelige klasser og selektorer

| Element | Selektor |
|---------|----------|
| Rot | `.ix-date-field` |
| Tekstfelt-wrapper | `.ix-text-field` |
| Synlig input | `.ix-date-field .ix-text-field > input` |
| Kalenderknapp (generert) | `.ix-date-field__toggle` |
| Native date-input (generert) | `.ix-date-field__native` |
| Label | `.ix-label` |
| Hjelpetekst | `[data-field="description"]` |
| Feilmelding | `[data-field="error"]` |

### `<ix-date-field>`

| Attributt | Påkrevd | Beskrivelse |
|-----------|---------|-------------|
| `data-open-label` | Ja | aria-label på den genererte kalenderknappen (i18n) |
| `name` | Nei | Navn på det innsendte feltet. Flyttes til den native ISO-inputen |
| `value` | Nei | Startverdi i ISO (`åååå-mm-dd`). Seeder synlig + native input |
| `min` | Nei | Tidligste valgbare dato i ISO. Speiles til native input |
| `max` | Nei | Seneste valgbare dato i ISO. Speiles til native input |
| `disabled` | Nei | Deaktiverer feltet |
| `readonly` | Nei | Skrivebeskyttet felt |
| `data-state` | Nei | `error` / `readonly` / `disabled` — visuell tilstand |

### Delelementer (light DOM)

| Del | Påkrevd | Beskrivelse |
|-----|---------|-------------|
| `.ix-text-field > input` | Ja | Synlig input. Skal ha `data-format="date"` og `inputmode="numeric"` |
| `.ix-date-field__toggle` | Nei | Kalenderknapp. Genereres automatisk om den mangler |
| `.ix-date-field__native` | Nei | Native `<input type="date">`. Genereres automatisk om den mangler |
