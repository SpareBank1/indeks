# Icon

Ikon-komponenten viser SVG-ikoner fra SpareBank 1s CDN ved hjelp av CSS `mask-image`. Ikoner arver farge fra omgivende tekst.

## Egnet til

- Dekorative ikoner ved siden av tekst
- Semantiske ikoner som formidler informasjon uten tilstΟΗtende tekst (med `ariaLabel`)

## Uegnet til

- Ikoner som eneste interaktive element uten tilgjengelig navn βÄî bruk en knapp med `aria-label` rundt ikonet

## Retningslinjer

### Bruk ikon sammen med tekst der det er mulig

Ikoner alene er tvetydige. Kombinasjon med synlig tekst gjΟΗr intensjonen klar for alle brukere, uavhengig av kjennskap til ikonsprΟΞket.

### Dekorative ikoner trenger ingen label

Ikoner som stΟΗtter tekst som allerede forklarer innholdet er dekorative. Ikke legg til `ariaLabel` βÄî det gir dobbel annonsering til skjermlesere.

### MeningsbΟΠrende ikoner trenger aria-label

### Ikoner i knapper og lenker

### Farge er ikke alene bΟΠrer av mening

Ikonet skal kommunisere gjennom form, ikke farge alene. Brukere med fargeblindhet eller hΟΗy kontrast-modus skal forstΟΞ ikonet uten ΟΞ se fargen.

### Velg stΟΗrrelse etter kontekst

## UnngΟΞ "blink" fΟΗr web componenten laster

Ikonet tegnes av CSS via `mask-image: var(--ii-icon-url)`. Det er web componenten som setter `--ii-icon-url` i sin `connectedCallback`. Frem til den kjΟΗrer er variabelen ikke satt, og CSS faller tilbake til en gjennomsiktig maske βÄî ikonet er da usynlig, ikke en farget boks.

```html live

```

:::note Hacky, men trygt som fallback
`--ii-icon-url` er en intern variabel (`--ii-`-prefiks). Den er ikke et offisielt API og kan i teorien endres. Bruk dette kun som en bevisst optimalisering for ikke-React-sider der blinket er et problem βÄî for `name`-aliaser mΟΞ du selv slΟΞ opp Material Design-filnavnet (f.eks. `hjem` βÜ£ `home.svg`), mens `materialdesignname`-verdien er filnavnet direkte.
