# Surface

Surface er en enkel visuell flate som grupperer relatert innhold, gjerne med en surface-farge,
border eller avrundede hjørner. Den er bevisst enkel: en ren beholder for layout og visuell
gruppering — uten elevation, klikk-affordanse eller annen interaktivitet.

Surface bygger i sin helhet på [utility-klasser](../utility-klasser/oversikt), men gir et React-API
slik at du slipper å forholde deg til klassenavnene direkte.

## Egnet til

- Visuell gruppering av relatert innhold i en flate med farge eller border
- Seksjoner og paneler i et oppsett (sett `as="section"` for riktig semantikk)
- Et lett alternativ til `Card` når du ikke trenger et avgrenset, klikkbart innholdskort
- Rask layout med `gap`, `justifyContent` og `alignItems` uten egen CSS

## Uegnet til

- Klikkbare flater eller navigasjon — bruk [Card](./card) med `href`/`onClick`
- Innhold som skal fremstå som en avgrenset enhet med løft/skygge — bruk [Card](./card)
- Komplekse layouter med retning og innrykk — vurder [Stack](./primitives/stack)

## Surface vs. Card

Surface og [Card](./card) løser ulike behov:

- **Surface** er en *enkel flate*. Den signaliserer ingen interaktivitet og har ingen elevation —
  den grupperer innhold visuelt. Tenk bakgrunn, seksjon eller panel.
- **Card** er et *innholdskort*. Det kan være klikkbart, bærer skygge og chevron når det er det, og
  fremstår som en avgrenset enhet.

Trenger du at flaten kan trykkes, eller at den skal «løftes» fra siden, velg Card. Ellers Surface.

## Eksempler

### Surface-farger

```tsx live
<VStack gap="md">
  <Surface surfaceColor="main" radius="md" padding="md">Main</Surface>
  <Surface surfaceColor="info" radius="md" padding="md">Info</Surface>
  <Surface surfaceColor="success" radius="md" padding="md">Suksess</Surface>
  <Surface surfaceColor="warning" radius="md" padding="md">Advarsel</Surface>
  <Surface surfaceColor="danger" radius="md" padding="md">Fare</Surface>
</VStack>
```

### Avrundede hjørner (radius)

`radius` mapper til `ix-radius-*`-utility og finnes i størrelsene `xs`, `sm`, `md`, `lg`, `xl`,
`pill` og `circle`.

```tsx live
<HStack gap="md">
  <Surface surfaceColor="info" radius="sm" padding="md">sm</Surface>
  <Surface surfaceColor="info" radius="md" padding="md">md</Surface>
  <Surface surfaceColor="info" radius="lg" padding="md">lg</Surface>
</HStack>
```

### Border

```tsx live
<HStack gap="md">
  <Surface border="default" radius="md" padding="md">default</Surface>
  <Surface border="dashed" radius="md" padding="md">dashed</Surface>
</HStack>
```

### Layout med flex-props

Surface har `gap`, `justifyContent` og `alignItems` for rask layout uten egen CSS.

```tsx live
<Surface
  surfaceColor="main"
  border="default"
  radius="md"
  padding="md"
  gap="sm"
  justifyContent="space-between"
>
  <Text>Venstre</Text>
  <Text>Høyre</Text>
</Surface>
```

## Props / API

| Prop | Type | Standard | Beskrivelse |
|------|------|----------|-------------|
| `as` | `ElementType` | `'div'` | Hvilket HTML-element flaten rendres som (f.eks. `'section'`) |
| `surfaceColor` | `SurfaceColor` | | Bakgrunnsfarge: `main` / `accent` / `info` / `success` / `warning` / `danger` / `interactive` |
| `border` | `'default' \| 'dashed'` | | Kantlinjestil |
| `radius` | `Radius` | | Avrundede hjørner: `xs` / `sm` / `md` / `lg` / `xl` / `pill` / `circle` |
| `padding` | `Size` | | Indre luft (også `paddingX`/`paddingY`/`paddingTop` osv.) |
| `margin` | `Size` | | Ytre luft (også `marginX`/`marginY` osv.) |
| `gap` | `Size` | | Avstand mellom barn (flex) |
| `justifyContent` | `'start' \| 'end' \| 'center' \| 'space-between'` | | Justering langs hovedaksen |
| `alignItems` | `'start' \| 'end' \| 'center' \| 'baseline'` | | Justering på tvers |
| `fullWidth` | `boolean` | `false` | Strekker flaten til full bredde |
| `className` | `string` | | Ekstra klasser |

I tillegg støttes standard HTML-attributter, som settes direkte på det rendrede elementet.

## Tilpasning med CSS

Trenger du flaten uten React-komponenten, bruk klassene direkte. Surface er i praksis bare
`ix-surface` kombinert med utility-klasser:

| Egenskap | Klasse |
|----------|--------|
| Flate (base) | `.ix-surface` |
| Surface-farge | `.ix-color-surface-{variant}-default` |
| Border | `.ix-border-default` / `.ix-border-dashed` |
| Radius | `.ix-radius-{xs\|sm\|md\|lg\|xl\|pill\|circle}` |
| Spacing | `.ix-p-*` / `.ix-m-*` |
| Gap | `.ix-gap-*` |

```html
<section class="ix-surface ix-color-surface-info-default ix-radius-md ix-p-md">
  <h3>Tittel</h3>
  <p>Innhold</p>
</section>
```

## Universell utforming

Surface er en ikke-interaktiv container og tilfører ingen rolle eller fokus-håndtering selv.

- **Velg semantisk element** med `as` der det er relevant — f.eks. `as="section"` for en seksjon.
  Standard `<div>` er korrekt når flaten kun er visuell gruppering.
- **Ikke kommuniser mening med farge alene** — surface-farge er dekorativ. Innhold som bærer
  betydning må også ha tekst eller annet ikke-visuelt signal.
- **Trenger flaten å være klikkbar**, er Surface feil verktøy — bruk [Card](./card), som gir
  riktig semantikk (`<a>`/`<button>`), tastaturstøtte og fokusring.
