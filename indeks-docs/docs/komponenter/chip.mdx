# Chip

Chip er små, interaktive komponenter som lar brukeren gjøre raske valg, filtrere innhold eller trigge handlinger. Chips brukes ofte i grupper og gir en kompakt og oversiktlig måte å presentere valg på.

Denne siden dokumenterer **button chip** — en chip som fungerer som en knapp og trigger en handling uten vedvarende valgt tilstand — og **removable chip** — en chip som representerer et aktivt valg og kan fjernes.

## Egnet til

- Hurtigvalg og enkle handlinger
- Forslag brukeren kan trykke på (f.eks. forslag til søk eller beløp)
- Handlinger med lav kompleksitet, presentert kompakt
- Grupper av relaterte handlinger

## Uegnet til

- Vedvarende valg der tilstanden skal huskes — bruk radio- eller checkbox-chip (kommer)
- Kritiske eller irreversible handlinger — bruk en tydelig knapp med bekreftelse
- Lange tekster eller forklaringer
- Den primære handlingen i en visning — bruk Button

## Removable chip

En removable chip representerer et aktivt valg eller en verdi som brukeren kan fjerne — for eksempel et valgt filter. Den finnes kun i valgt tilstand (den eksisterer ikke før den er valgt) og vises med fylt aksentfarge og et kryss til høyre.

**Hele chipen er fjern-knappen.** Et klikk hvor som helst på chipen — eller Enter/Mellomrom når den har fokus — fjerner den. Krysset er et dekorativt CSS-ikon uten eget fokus, så det blir kun ett tab-stopp per chip. Dette følger mønsteret i Aksel og Designsystemet.

Det tilgjengelige navnet settes sammen av label og et fjern-suffiks, f.eks. «Sparing fjern». I React sender du suffikset via `removeLabel`; i ren HTML setter du hele `aria-label` selv. Suffikset må være på brukerens språk — komponenten har ingen hardkodet fallback.

<Tabs groupId="variant">
<TabItem value="html" label="HTML">

```jsx live
<button class="ix-chip" data-removable aria-label="Sparing fjern">
  Sparing
</button>
```

</TabItem>
<TabItem value="react" label="React">

```tsx live
<RemovableChip removeLabel="fjern" onRemove={() => {}}>
  Sparing
</RemovableChip>
```

</TabItem>

## Retningslinjer

### Bruk chip for raske og enkle valg

Chips egner seg for enkle interaksjoner der brukeren raskt skal kunne velge, filtrere eller utføre en handling. De fungerer best når valgene er korte, tydelige og enkle å forstå.

### Teksten skal være kort og tydelig

Chips skal ha korte labels som er enkle å skanne. Unngå lange tekster — chips er ment for raske valg, ikke forklaringer.

### Chips bør brukes i grupper

Chips gir mest verdi når de presenteres sammen i en gruppe. Enkeltstående chips kan brukes, men da primært som button chip for å trigge en handling.

### Unngå for mange chips samtidig

Store mengder chips kan bli vanskelig å skanne og bruke. Dersom det er mange valg, bør de grupperes eller presenteres på en annen måte.

### Bruk ikon med omtanke

Ikoner kan støtte forståelsen, men skal ikke erstatte tekst. Tekst er alltid den viktigste informasjonsbæreren.

## Universell utforming

### Hva du selv må sørge for

- **Gi chipen en kort og tydelig label** — teksten er det tilgjengelige navnet. Den skal beskrive valget eller handlingen.
- **Bruk chips i grupper** — en enkeltstående chip gir lite kontekst. Presenter relaterte chips sammen.
- **Ikke stol på farge alene** — teksten må alltid være den primære informasjonsbæreren.
- **Bruk riktig element** — `<button>` (standard) for handlinger og `as="a"` / `<a>` for navigasjon. Ikke legg click-handlers på `<div>` eller `<span>`.
- **Gi removable chip et tilgjengelig fjern-navn** — send `removeLabel` (React) eller sett hele `aria-label` (HTML), f.eks. «Sparing fjern». Uten det hører skjermleserbrukeren bare «Sparing, knapp» og vet ikke at klikk fjerner den. Bruk riktig språk (bokmål «fjern», nynorsk «fjern», engelsk «remove») — ingen hardkodet fallback.
- **Unngå disabled som standard** — en disabled chip sier ikke hva som mangler. La heller chipen være aktiv og forklar konteksten.

### Hva komponenten gjør automatisk

- **`type="button"`** settes som standard på `<button>`-elementer så chipen ikke sender inn skjema utilsiktet.
- **Tilgjengelig navn** hentes fra tekst-innholdet i chipen. For removable chip settes `aria-label` til «[label] [removeLabel]».
- **Fokusindikator** via `:focus-visible` — synlig for tastaturbrukere, ikke for mus.
- **Tilstand med fyll og kant** — hover og active kommuniseres med mer enn farge.
- **Fjern-krysset er dekorativt** — det rendres med CSS (`::after`) og ligger ikke i tilgjengelighetstreet. Hele removable-chipen er fjern-knappen, så det blir kun ett tab-stopp.

## Props / API

### `Chip` (button chip)

| Prop | Type | Standard | Beskrivelse |
|------|------|----------|-------------|
| `size` | `'sm' \| 'md'` | `'md'` | Størrelse |
| `as` | `ElementType` | `'button'` | Render som annet element, f.eks. `'a'` for lenke-chips |
| `disabled` | `boolean` | | Deaktiverer chipen (nativ HTML-attributt) |

### `RemovableChip`

| Prop | Type | Standard | Beskrivelse |
|------|------|----------|-------------|
| `children` | `string` | | Etiketten på chipen. Inngår i det tilgjengelige navnet |
| `removeLabel` | `string` | | **Påkrevd.** Suffiks i det tilgjengelige navnet, f.eks. `"fjern"` gir «Sparing fjern». i18n — ingen fallback |
| `onRemove` | `() => void` | | Kalles når chipen fjernes (klikk / Enter / Mellomrom) |
| `size` | `'sm' \| 'md'` | `'md'` | Størrelse |
| `disabled` | `boolean` | | Deaktiverer chipen (nativ HTML-attributt) |

I tillegg støttes alle standard HTML-knapp-attributter (`onClick`, `onFocus`, `type`, `name`, `value`, `tabIndex` osv.) og settes direkte på elementet.

## Tilpasning med CSS

Trenger du `ix-chip`-stylingen på HTML du setter sammen selv — uten React-komponenten — kan du bruke klassen direkte på elementene dine.

### Tilgjengelige klasser og attributter

| Element/tilstand | Selektor / attributt |
|-----------------|----------------------|
| Chip | `.ix-chip` |
| Liten størrelse | `data-size="sm"` |
| Removable chip (valgt) | `data-removable` |
| Fjern-kryss | `.ix-chip[data-removable]::after` (dekorativt, ingen markup) |
| Hover / active | `:hover` / `:active` (automatisk) |
| Fokus | `:focus-visible` (automatisk) |
| Deaktivert | `disabled` eller `aria-disabled="true"` |

### Eksempel: ren HTML

```html
<button class="ix-chip" type="button">
  Chip label
</button>

<button class="ix-chip" data-size="sm" type="button">
  Liten chip
</button>

<a class="ix-chip" href="/sparing">
  Sparing
</a>

<button class="ix-chip" data-removable aria-label="Sparing fjern">
  Sparing
</button>
```
